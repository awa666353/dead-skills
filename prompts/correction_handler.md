# 对话纠正处理器（逝者 Skill）

## 任务

当 **制作者或核心家属** 在使用或调试逝者 Skill 时表达「不对」「TA 不会这样说」「事实应该是…」，识别纠正意图并更新 `life.md` 或 `persona.md`，流程与 yourself-skill 的 correction_handler 对齐，对象从「我」转为「逝者/材料叙述」。

---

## 触发识别

以下表达触发纠正模式（示例）：

- 「不对」「不是这样的」「记错了」
- 「他不会这样说」「她不会这么说话」
- 「应该是…」「其实那一年…」「没这回事」
- 「太温柔了」「太硬了」「太文艺了，不像他」
- 「他遇到这种情况会…」

---

## 纠正分类

### Life Memory 纠正（事实/叙事类）

- 「他从没在那单位待过」→ 改时间线/职业
- 「他不吃辣」→ 改饮食习惯
- 「最常去的不是公园，是河边钓台」→ 改空间偏好

### Persona 纠正（口吻/性格类）

- 「他不会用这么多 emoji」→ 改 Layer 2
- 「他生气不会冷战这么久」→ 改 Layer 3
- 「他从不会主动跟邻居打招呼」→ 改 Layer 4

---

## 处理流程

1. **确认纠正内容**：向用户确认理解是否正确。

   ```
   我理解了：不是说「{旧行为}」，而是「{新行为}」，对吗？
   ```

2. **生成 Correction 记录**：

   ```markdown
   ### Correction #{n} — {日期}
   - 文件：life.md / persona.md
   - 层级或章节：{Layer X / 某节标题}
   - 原文：{被纠正的描述}
   - 纠正为：{新的描述}
   - 纠正人原话：「{用户表述}」
   ```

3. **追加**到对应文件末尾的 `## Correction 记录` 节（若尚无该节则创建）。

4. **同步修改正文**：将被替换的原文更新为新表述；可在旁标注 `[已纠正，见 Correction #n]`。

5. **重新生成 SKILL.md**：

   ```bash
   python ${CLAUDE_SKILL_DIR}/tools/skill_writer.py --action combine --slug {slug} --base-dir ./.claude/skills
   ```

6. **更新 `meta.json`**：`corrections_count` +1，`updated_at` 刷新。

---

## 注意事项

- **不质疑家属**对事实的最终说法（可请求二次确认避免听错）。
- **第三人隐私**：纠正中若涉及在世者敏感信息，写入时脱敏或概括。
- 纠正后 **下一条对话** 即应按新规则体现。
- 与 yourself-skill 相同：**纠正优先于旧模型惯性**。
