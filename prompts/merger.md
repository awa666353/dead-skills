# 增量 Merge 逻辑（逝者 Skill）

## 任务

当用户追加新的原材料（悼文定稿、新找到的聊天节选、家书电子版、照片说明等）时，将增量信息 merge 进现有的 `life.md` 和 `persona.md`，**默认不覆盖**已有结论，结构与 yourself-skill 的 merger 对齐，仅文件名由 `self.md` 换为 `life.md`。

---

## 原则

1. **增量不覆盖**：新信息追加到已有内容之后，或插入时间线正确位置；不无故删除旧段落。
2. **冲突标注**：若新信息与已有叙述矛盾，用 `[⚠️ 冲突]` 标出，并请家属择一或并存备注。
3. **时间线补充**：新事件按时间顺序并入「人生时间线」表格。
4. **证据升级**：若新材料显著加强某条结论，可在该条后附 `（补证于 {日期}）`。
5. **转述标注**：纯口述回忆宜加 `（转述：某某）` 以便溯源。

---

## 流程

### 1. 分析增量内容

将新材料按 `life_analyzer.md` 与 `persona_analyzer.md` 的维度分析，输出：

- 新的生平节点与价值观证据
- 新的性格/口吻证据
- 与现有内容的一致性 / 冲突点

### 2. Merge 策略

**Life Memory（`life.md`，事实与叙事类）**：

- 新经历 → 插入「人生时间线」对应行或追加新表
- 新价值观证据 → 追加到「价值观与看重的事」各小节底部
- 新习惯 → 追加到「生活习惯」
- 人际关系新信息 → 追加到「人际关系」对应部分
- 新被转述话语 → 追加到「精神遗产与话语」

**Persona（`persona.md`，口吻与行为类）**：

- 新口头禅、新示例句 → 追加到 Layer 2「示例表达」或「语言习惯」
- 更充分的情感模式证据 → 强化 Layer 3 描述（保留原句可加删除线或由家属定夺）
- 新行为模式 → 追加到 Layer 4
- **Layer 0 / Layer 1 通常不变**（除非伦理或身份事实有纠正）

### 3. 输出与标注

用 `Edit` 工具在对应章节写入增量，并在新内容前标注：

```markdown
<!-- [追加于 {日期}，来源：{来源类型}] -->
{新内容}
```

### 4. 后处理

- 运行 `skill_writer.py --action combine --slug {slug} --base-dir ./.claude/skills`
- 更新 `meta.json` 的 `updated_at`、`memory_sources`、`version`（按项目版本惯例）

---

## 与 yourself-skill 的差异

|  yourself-skill | 逝者 Skill |
|----------------|------------|
| `self.md` | `life.md` |
| 「自我记忆」章节名 | 「生平与记忆」等（见 `life_builder.md`） |
| 用户即本人 | 用户多为家属；merge 时注意 **第三人称口吻** |
